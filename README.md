# Лабораторная работа 2: LLM и Трансформеры

Выполнена в рамках дисциплины Deep Learning (ITMO)

---

## 1. Назначение и цель работы

Изучить архитектуру трансформеров, освоить методыfine-tuning предобученных моделей (GPT, BERT) для задачи генерации текста, проанализировать влияние различных оптимизаторов и scheduler'ов на качество обучения.

**Целевые задачи:**
- Подготовка текстового корпуса для обучения
- Изучение архитектуры трансформера (Self-Attention, Multi-Head Attention, Feed-Forward Network)
- Fine-tuning модели трансформера
- Оценка качества с помощью метрик Perplexity, BLEU, ROUGE
- Разработка чат-бота для интерактивного тестирования

---

## 2. Этапы выполнения

### 2.1. Этап 1: Введение и подготовка данных

**Задачи:**

| № | Задача |
|---|--------|
| 1 | Подготовить текстовый корпус для обучения модели |
| 2 | Разделить данные на тренировочный, валидационный и тестовый наборы |
| 3 | Очистить данные: удаление пунктуации, стоп-слов, лемматизация |
| 4 | Преобразовать текст в числовое представление через токенизацию |

**Используемые инструменты:**
- `nltk` / `spacy` — лемматизация, удаление стоп-слов

**Выходные артефакты:**
- Очищенный и токенизированный корпус
- train/val/test сплиты

### 2.2. Этап 2: Изучение архитектуры трансформера

**Теоретические вопросы для изучения:**

| Компонент | Описание |
|-----------|----------|
| Self-Attention | механизм взвешивания важности токенов в последовательности |
| Multi-Head Attention | параллельные механизмы внимания для捕捉 различных связей |
| Feed-Forward Network | полносвязный слой после блока внимания |
| Positional Encoding | добавление информации о позиции токена |
| Residual Connections | обходные соединения для улучшения сходимости |

**Ключевые преимущества трансформера перед RNN/LSTM:**
- Параллельная обработка последовательности
- Отсутствие проблем затухающих/взрывающихся градиентов
- Лучший захват долгосрочных зависимостей
- Масштабируемость архитектуры

**Выходные артефакты:**
- Краткое описание компонентов в отчёте
- Схема архитектуры трансформера (опционально)

### 2.4. Этап 4: Обучение модели

**Задачи:**

| № | Задача |
|---|--------|
| 1 | Провести обучение трансформера на подготовленных данных |
| 2 | Применить различные оптимизаторы и сравнить их влияние |
| 3 | Использовать learning rate scheduler |

**Исследуемые оптимизаторы:**

| Оптимизатор | Особенности |
|-------------|-------------|
| Adam | адаптивный, наиболее распространённый |
| SGD | классический стохастический градиентный спуск |
| RMSProp | адаптивный, хорош для нестационарных задач |

**Learning rate schedulers:**

| Scheduler | Описание |
|-----------|----------|
| Linear | линейное уменьшение LR |
| Cosine | косинусоидальное затухание |
| Step | ступенчатое уменьшение |
| Exponential | экспоненциальное затухание |

**Выходные артефакты:**
- Сохранённые чекпоинты моделей
- Логи обучения (loss, perplexity)

### 2.5. Этап 5: Оценка модели

**Метрики оценки:**

| Метрика | Назначение | Формула / принцип |
|---------|------------|--------------------|
| **Perplexity** | оценка уверенности модели | `exp(loss)`, чем ниже, тем лучше |
| **BLEU** | качество генерации текста | сравнение n-грамм с эталоном |
| **ROUGE** | качество генерации текста | полнота n-грамм, особенно для суммаризации |

**Сравнительный анализ:**

| Условие | Perplexity | BLEU | ROUGE |
|---------|------------|------|-------|
| Adam + Linear | | | |
| Adam + Cosine | | | |
| SGD + Step | | | |
| RMSProp + Exponential | | | |

**Выходные артефакты:**
- Таблица сравнения метрик
- Графики зависимости Perplexity от эпох

## 6. Примечания

- В качестве базовой модели рекомендуется использовать **GPT-2** (small или medium) для задач генерации.
- Для ускорения экспериментов допускается использование **DistilGPT-2**.
- Fine-tuning выполняется на русскоязычном или англоязычном корпусе (в зависимости от задачи).
- При обучении на ограниченных ресурсах рекомендуется использовать градиентное накопление.
